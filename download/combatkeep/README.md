# CombatKeep - مود ماينكرافت Fabric 26.1.2

## 🎮 نبذة عن المود

CombatKeep هو مود من جهة السيرفر فقط (Server-Side) لماينكرافت إصدار 26.1.2 يعمل على Fabric Loader.
يقوم المود بتفعيل KeepInventory تلقائياً مع إضافة نظام قتال عقابي يعتمد على التاق (Combat Tag).

## ✨ المميزات

### نظام KeepInventory
- يتم تفعيل `keepInventory` تلقائياً على جميع العوالم
- لا يمكن إيقافه حتى لو حاول المشرفون تغييره (يتم إعادة تفعيله تلقائياً)

### نظام الموت العادي
- عند الموت خارج القتال: يخسر اللاعب **نقاط الخبرة فقط** (XP)
- يحتفظ اللاعب بجميع أغراضه في المخزون

### نظام Combat Tag (تاق القتال)
- عندما يضرب لاعب آخر لاعب، يتم تفعيل تاق قتال لمدة **15 ثانية**
- التاق ينطبق على **كلا اللاعبين** (الضارب والمضروب)
- العداد **يتجدد** في كل مرة يضرب فيها أحد اللاعبين الآخر
- يظهر **Boss Bar** في أعلى الشاشة يعرض الوقت المتبقي للقتال
- لون Boss Bar يتغير:
  - 🔴 **أحمر**: أكثر من 7 ثوانٍ متبقية
  - 🟡 **أصفر**: 7 ثوانٍ أو أقل
  - ⚪ **أبيض وامض**: 3 ثوانٍ أو أقل (حرج!)

### عقوبة الموت أثناء القتال
- يخسر اللاعب **جميع نقاط الخبرة** (XP)
- يخسر اللاعب **50% من أغراض المخزون** (تسقط على الأرض في مكان الموت)
- الأغراض تُختار **عشوائياً** من المخزون
- **مستثنى من السقوط**:
  - 🛡️ الدروع التي يرتديها اللاعب (خانات 36-39)
  - 📦 جميع أنواع الشلكر بوكس (Shulker Boxes) بألوانها المختلفة
- الأغراض المسقطة يمكن لأي لاعب التقاطها

### عقوبة الخروج أثناء القتال (Combat Log)
- إذا خرج اللاعب من السيرفر أثناء وجود تاق قتال نشط
- يُعامل كأنه **استسلم من القتال**
- تُطبق نفس عقوبة الموت أثناء القتال:
  - خسارة جميع نقاط الخبرة
  - خسارة 50% من الأغراض (تسقط في مكان وقوفه)
  - الدروع والشلكر بوكس مستثناة

## 🔧 التثبيت

### على السيرفر
1. تأكد من تثبيت Fabric Loader 0.19.2+ على السيرفر
2. حمّل Fabric API لإصدار 26.1.2 من [Modrinth](https://modrinth.com/mod/fabric-api) أو [CurseForge](https://www.curseforge.com/minecraft/mc-mods/fabric-api)
3. ضع ملف `combatkeep-1.0.0.jar` وملف Fabric API في مجلد `mods` بالسيرفر
4. أعد تشغيل السيرفر

### على العميل
**لا حاجة لتثبيت المود على جانب العميل!** المود يعمل من جهة السيرفر فقط.

## 🧠 كيفية عمل المود

### تدفق أحداث القتال
```
لاعب A يضرب لاعب B
    ↓
تفعيل Combat Tag لكلا اللاعبين (15 ثانية)
    ↓
عرض Boss Bar للاعبين
    ↓
┌─────────────────────────────────┐
│  إذا ضرب أحدهما الآخر مجدداً:  │
│  → تجديد العداد إلى 15 ثانية   │
└─────────────────────────────────┘
    ↓
┌──────────────┬───────────────────┐
│  الموت أثناء │  انتهاء العداد   │
│  القتال      │  بدون موت        │
├──────────────┼───────────────────┤
│  خسارة 50%   │  لا عقوبة        │
│  من الأغراض  │  Combat Tag       │
│  + كل XP     │  ينتهي بأمان     │
└──────────────┴───────────────────┘

┌──────────────────────────────────┐
│  الخروج من السيرفر أثناء القتال │
│  → نفس عقوبة الموت أثناء القتال │
└──────────────────────────────────┘
```

### آلية اختيار الأغراض المسقطة
1. يتم جمع جميع خانات المخزون القابلة للسقوط (بدون الدروع والشلكر بوكس)
2. يتم خلط القائمة بشكل عشوائي
3. يتم اختيار 50% من الخانات (مقربة لأعلى)
4. الأغراض المختارة تُسقط كـ Item Entities في مكان الموت
5. الأغراض المتبقية تبقى مع اللاعب

## ⚙️ إعدادات تقنية

- **نوع المود**: Server-Side Only (لا يحتاج تثبيت على العميل)
- **Fabric Loader**: 0.19.2+
- **Loom Plugin**: `net.fabricmc.fabric-loom` (الإصدار الجديد لـ 26.1+)
- **Java Target**: 25
- **Minecraft غير مشفّر** (Unobfuscated) في إصدار 26.1.2

## 🔄 تغييرات إصدار 26.1.2 المهمة للمطورين

- ماينكرافت 26.1 **غير مشفّر** (Unobfuscated) - لا حاجة لـ mappings
- اسم plugin لـ Loom تغير من `fabric-loom` إلى `net.fabricmc.fabric-loom`
- `modImplementation` أصبح `implementation`
- `remapJar` أصبح `jar`
- Java 25 مطلوب بدلاً من Java 21
- حقل `mappings` في `dependencies` تم إزالته
